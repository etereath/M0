# 架构决策记录（ADR 摘要）

## ADR-001：使用前缀 ULID

**决定：** 业务实体 ID 使用前缀 ULID。  
**理由：** 本地和云端都可离线生成，按时间大致排序，减少中心化 ID 服务依赖。

## ADR-002：逾期不是任务状态

**决定：** `is_overdue` 为派生属性。  
**理由：** 任务可以同时处于 `BLOCKED` 或 `NEEDS_REVIEW` 并且逾期，单一状态无法表达两个维度。

## ADR-003：追加事件不可修改

**决定：** 库存流水、任务更新和审计事件不可原地修改。  
**理由：** 保留可追溯性，并简化云本地同步。

## ADR-004：项目保留确认进度和建议进度

**决定：** `progress_percent_confirmed` 与 `progress_percent_suggested` 分离。  
**理由：** Agent 可以计算建议，但不能替负责人确认。

## ADR-005：冲突不使用最后写入覆盖

**决定：** 双边主档冲突进入人工队列。  
**理由：** 当前数据规模小，人工解决成本低于静默覆盖造成的业务风险。
