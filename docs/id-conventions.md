# ID、时间、版本和删除约定

## 1. 实体 ID

统一格式：`<PREFIX>_<ULID>`。ULID 使用 Crockford Base32，26 位大写字符，排除 I、L、O、U。
ID 由首次创建实体的一方生成，云端与本地同步时保持不变。

| 实体 | 前缀 |
|---|---|
| 员工 | EMP |
| 任务 | TASK |
| 分配 | ASGN |
| 任务反馈 | TUPD |
| 项目 | PROJ |
| 项目更新 | PUPD |
| 里程碑 | MILE |
| 物料 | ITEM |
| 批次 | LOT |
| 库存流水 | ITXN |
| Agent 提案 | APRP |
| 审批请求 | APRV |
| 同步事件 | SYNC |
| 审计事件 | AUDT |

示例：`TASK_01J2K9QX4R8S6B7C5D3E1F0GHT`。

## 2. 时间

- API 输入输出：RFC 3339，例如 `2026-07-15T09:00:00Z` 或 `2026-07-15T17:00:00+08:00`；
- 持久化：统一转为 UTC；
- 业务展示默认时区：`Asia/Shanghai`；
- 禁止无时区时间；
- 仅日期字段使用 `YYYY-MM-DD`。

## 3. 版本和哈希

- 可修改主档：整数 `version`，从 1 开始，每次成功修改 +1；
- 客户端更新主档必须携带期望版本；版本不符返回 `CONFLICT_VERSION_MISMATCH`；
- `content_hash` 使用规范化 JSON 的 SHA-256，格式 `sha256:<64 hex>`；
- 追加事件不得更新内容；纠错通过新增 `CORRECTION` 事件。

## 4. 删除

- 业务主档默认软删除：设置 `deleted_at`；
- 库存流水、任务反馈、审批结果、同步事件和审计事件禁止删除；
- 飞书侧误删时，本地应记录审计事件并进入冲突处理，不自动物理删除本地事实。
